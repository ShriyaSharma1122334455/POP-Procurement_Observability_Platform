# Product

## Register

product

## Users

Procurement managers, CFOs, operations managers, and restaurant owners who need real-time visibility into procurement spend. They work inside the platform frequently — viewing dashboards, monitoring suppliers, responding to alerts, and querying the AI agent. Context is task-driven: check spend trends before a meeting, investigate an alert, ask the agent for savings ideas.

## Product Purpose

POP (Procurement Observability Platform) converts procurement from a reactive, spreadsheet-driven process into an observable system — the Datadog for procurement. It continuously ingests purchasing data and surfaces actionable intelligence: where money is going, why spend changed, which suppliers are underperforming, what savings exist, and what risks need attention.

## Brand Personality

Analytical. Trustworthy. Decisive.

The platform handles real financial data; it must feel authoritative and precise, not playful. Confidence without arrogance. Clarity without sterility.

## Anti-references

- Consumer dashboards that prioritize visual wow over data legibility
- Generic SaaS "feature grids with icons" aesthetics
- Overly dark "hacker terminal" UIs that feel hostile
- Cream/paper/warm-tinted backgrounds that feel like a blog

## Design Principles

1. **Data over decoration** — every visual element earns its place by making information clearer or actions easier.
2. **Earned familiarity** — borrow conventions from tools users already trust (Linear, Notion, Stripe dashboard) so the interface disappears into the task.
3. **Severity is visible** — alerts, risk scores, and changes should be scannable at a glance without reading labels.
4. **Progressive detail** — summary first, drill-down on demand. Never surface everything at once.
5. **Consistent vocabulary** — the same component does the same thing everywhere. No creative reinterpretation of standard affordances.

## Accessibility & Inclusion

WCAG AA minimum. All interactive elements keyboard-accessible. Color is never the only indicator of state (severity badges use icon + color + text). Support `prefers-reduced-motion`.
